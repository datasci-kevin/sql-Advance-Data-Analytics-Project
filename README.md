## 📌 Project Overview

This project is a complete **Exploratory Data Analysis (EDA)** workflow built with SQL Server.  
The goal of this project is to analyze a sales dataset from multiple business angles and transform raw tables into clear, structured insights.

Rather than writing disconnected queries, this project follows a professional analytical process:

1. Understand the database structure  
2. Explore dimensions and measures  
3. Analyze sales trends over time  
4. Identify top-performing customers and products  
5. Compare performance across years  
6. Segment customers and products for deeper insights  
7. Build final SQL-based reports  

This project demonstrates my ability to think like a data analyst while also organizing SQL scripts in a clean, reusable, and portfolio-ready format.

## 🎯 Business Questions Answered

This analysis focuses on answering questions such as:

- How much historical data is available in the database?
- What are the main dimensions and measures in the dataset?
- Which products and customers generate the most sales?
- How do sales change over time?
- What are the year-over-year performance trends?
- Which categories contribute the most to total sales?
- How can customers and products be segmented for business reporting?

## 🔍 Analysis Breakdown

### 1. Database Exploration
The first step focuses on understanding the structure of the database, including available tables, columns, and relationships.

### 2. Dimension Exploration
This section explores categorical fields such as customers, products, countries, categories, and other descriptive attributes.

### 3. Date Range Exploration
This step identifies how much historical data is available and determines the time period covered by the dataset.

### 4. Measure Exploration
This section reviews key numeric fields such as sales amount, quantity, cost, and other measurable values.

### 5. Magnitude Analysis
This analysis looks at business size and volume by comparing totals across categories, products, customers, and regions.

### 6. Ranking Analysis
This section identifies top and bottom performers using ranking logic, helping highlight the most important customers and products.

### 7. Change Over Time
This section analyzes how business performance changes across years and months.

### 8. Cumulative Analysis
This step calculates running totals and moving averages to understand long-term growth patterns.

### 9. Performance Analysis
This section compares current performance against previous periods using SQL window functions such as `LAG()`.

### 10. Part-to-Whole Analysis
This analysis measures how much each category, product, or segment contributes to the total.

### 11. Data Segmentation
This section groups customers and products into meaningful segments for easier business interpretation.

### 12. Customer Report
The customer report summarizes customer behavior, sales contribution, order patterns, and performance.

### 13. Product Report
The product report summarizes product performance, sales contribution, and ranking insights.
